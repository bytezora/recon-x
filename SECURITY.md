# Security Policy

## Authorized Use

recon-x is intended for authorized security testing, bug bounty work and internal defensive assessments. Do not scan assets without permission.

## Reporting Security Issues

If you find a vulnerability in recon-x itself, please open a private security advisory on GitHub or contact the maintainers through the repository owner profile.

Please include:

- affected version or commit
- operating system and Go version
- minimal reproduction steps
- expected and actual behavior
- impact and suggested remediation, if known

## Supported Versions

Security fixes are prioritized for the latest release and the `main` branch.

## Finding Validation

recon-x findings are indicators unless explicitly confirmed by a safe verification module. Validate impact manually before reporting issues to a third party.
